# Versions

## v0.0.0 -> v0.0.1

- Init repo

## v0.0.1 -> v0.0.2

- Создан стартовый skeleton проекта BBSim.
- Добавлен pipeline эпох с первыми stage-ами: personal seed, inflation, recombination preview.
- Добавлен NumPy backend для deterministic seed field и базовых операций с полями.
- Добавлен минимальный PySide6/PyQtGraph UI.
- Добавлены базовые тесты детерминизма seed-а, формулы расширения и порядка pipeline.

## v0.0.2 -> v0.0.3

- Синхронизирована версия проекта до 0.0.3 в package metadata, `__version__` и заголовке окна.
- Улучшена генерация личного seed field: вместо сырого белого шума используется сглаженное Fourier-shaped поле с крупномасштабной структурой и слабым fine layer.
- Переработаны метрики seed-а: крупномасштабный рисунок, мелкая зернистость, потенциал пустот и риск раннего коллапса считаются устойчивее.
- Добавлена человекочитаемая сигнатура зерна в checkpoint report.
- Добавлена космическая цветовая карта для центрального поля и нижний timeline текущих checkpoint-ов.
- Начальный масштаб `a` сдвинут глубже в раннюю эпоху, чтобы после инфляции preview-рекомбинация не определялась как поздняя dark-energy эпоха.

## v0.0.3 -> v0.0.4

- Добавлено отдельное поле `inflation_delta` для результата инфляционного сглаживания первичного seed field.
- `InflationStage` теперь формирует визуально отличимый checkpoint: подавляет мелкую рябь, сохраняет крупномасштабный рисунок и обновляет поля тёмной/обычной материи.
- Checkpoint инфляции получил дополнительные метрики: рост масштаба, подавление кривизны, сохранённый контраст и подавление мелкой ряби.
- CMB-preview теперь строится из инфляционно обработанного поля, а не напрямую из исходного зерна.
- График `a(t)` в UI переведён в `log10 a(t)`, чтобы был виден инфляционный скачок масштаба.
- Синхронизирована версия проекта до 0.0.4.

## v0.0.4 -> v0.0.5

- Добавлен TOML-конфиг приложения `config/app.toml` для UI-настроек.
- Размер и режим окна теперь читаются из конфига: `normal`, `maximized`, `fullscreen`.
- Центральная визуализация поля растягивается на весь canvas при `field_fill_canvas = true`.
- Скрыта служебная histogram-панель `ImageView`, чтобы поле занимало больше полезной площади.
- Синхронизирована версия проекта до 0.0.5.

## v0.0.5 -> v0.0.6

- Добавлен TOML-конфиг симуляции `config/simulation.toml` для seed, inflation, cosmology и prototype structure параметров.
- Добавлен загрузчик `load_simulation_config()` с поддержкой `BBSIM_SIMULATION_CONFIG`, CLI override и безопасной валидацией значений.
- Headless-режим теперь читает simulation config и позволяет передать `--simulation-config`; `--phrase` стал override поверх файла.
- В левую UI-панель добавлены базовые параметры инициализации: seed field, инфляция, H0, обычная материя, тёмная материя, тёмная энергия, радиация и кривизна.
- `Создать запуск` теперь собирает `UniverseConfig` из UI-панели, поэтому параметры реально влияют на seed, inflation и CMB-preview.
- Синхронизирована версия проекта до 0.0.6.

## v0.0.6 -> v0.0.7

- Нижний timeline заменён с маленькой текстовой подписи на отдельную полноширинную панель визуального прогресса.
- Добавлена модель `core.timeline` со списком основных эпох и расчётом позиции маркера без вывода числовых процентов в UI.
- Добавлен кастомный `TimelinePanel` на Qt/QPainter: подписи эпох, checkpoint-точки, заполненная линия пройденного пути и текущий бегущий маркер.
- Timeline теперь показывает весь будущий pipeline: зерно, инфляция, разогрев, нуклеосинтез, CMB, тёмные века, первые звёзды, галактики, тёмная энергия и судьба.
- Синхронизирована версия проекта до 0.0.7.

## v0.0.7 -> v0.0.8

- Стартовый GUI больше не создаёт seed field автоматически: приложение открывается в idle-состоянии без шума на центральном canvas.
- Центральная область до создания зерна показывает спокойный placeholder с инструкцией вместо первичной ряби.
- Правый график до создания run-а заменён placeholder-ом; отчёт показывает инструкцию ожидания.
- Кнопка `Создать запуск` переименована в `Создать зерно`; после первого создания становится `Пересоздать зерно`.
- `Следующий checkpoint` заблокирован до создания зерна и снова блокируется после завершения текущего pipeline, без повторяющихся сообщений `Pipeline завершён`.
- Timeline в idle-состоянии остаётся пустым, а активная позиция появляется только после создания зерна.
- Синхронизирована версия проекта до 0.0.8.

## v0.0.8 -> v0.0.9

- Добавлен live evolution loop в GUI: эпохи больше не перескакивают только по кнопке, а обновляются по `QTimer` во время наблюдения.
- Кнопочный flow заменён на одну главную кнопку `BIG BANG` / `ПАУЗА` / `ПРОДОЛЖИТЬ` / `НОВАЯ ВСЕЛЕННАЯ`.
- Добавлена настройка `pause_on_epochs` и UI-галочка `Останавливаться на эпохах`: checkpoint-и могут ставить симуляцию на паузу или проходиться без остановки.
- `PersonalSeedStage` теперь проявляет зерно визуально за короткую live-фазу, а не появляется мгновенно.
- `InflationStage` обновляет `inflation_delta` на каждом tick-е: поле непрерывно сглаживается и переходит от seed к инфляционному состоянию.
- `RecombinationPreviewStage` стал live-переходом к CMB-like отпечатку с обновлением температуры, `a(t)`, `H(a)` и центрального поля.
- Нижний timeline теперь двигает маркер во время текущей эпохи, а не только после checkpoint-а.
- Синхронизирована версия проекта до 0.0.9.

## v0.0.9 -> v0.0.10

- Добавлены live-stage-и ранней Вселенной между инфляцией и CMB: `ReheatingStage` и `NucleosynthesisStage`.
- Pipeline теперь честно проходит по нижнему timeline: зерно → инфляция → разогрев → нуклеосинтез → CMB, без прямого прыжка от инфляции к CMB.
- `ReheatingStage` формирует горячее radiation-поле на каждом tick-е, обновляет температуру, `a(t)`, `H(a)` и отчёт о радиационно-доминирующей плазме.
- `NucleosynthesisStage` показывает охлаждение горячей плазмы и формирует первичный состав: водород, гелий, следы лития и отсутствие тяжёлых элементов.
- `RecombinationPreviewStage` теперь строит CMB-like отпечаток из cooled radiation field, а не напрямую из post-inflation field.
- Добавлен `EarlyUniverseConfig` и секция `[early_universe]` в `config/simulation.toml` для длительности live-этапов разогрева, нуклеосинтеза и рекомбинации.
- Центральный canvas теперь показывает radiation-поле во время разогрева и нуклеосинтеза.
- Синхронизирована версия проекта до 0.0.10.

## v0.0.10 -> v0.0.11

- Добавлен `ExpansionEngine` как единая точка расчёта `H(a)`, density-компонентов radiation / matter / curvature / dark energy и доминирующей эпохи.
- `UniverseState` теперь хранит текущие плотности, display-доли компонентов и историю `H(a)`, температуры и компонент для графиков.
- Live-stage-и ранней Вселенной теперь записывают общие космологические сэмплы через `ExpansionEngine`, а не вручную обновляют только `a(t)`.
- Правая панель UI теперь показывает два графика: `log10 a(t)` и доли radiation / matter / dark energy.
- Отчёты разогрева, нуклеосинтеза и CMB получили строки с доминирующим компонентом и долями основных density-компонентов.
- Синхронизирована версия проекта до 0.0.11.

## v0.0.11 -> v0.0.12

- `RecombinationPreviewStage` заменён на полноценный live-stage `RecombinationStage`: CMB теперь появляется через падение ионизации и непрозрачности горячей плазмы.
- `UniverseState` получил параметры рекомбинации: `ionization_fraction`, `opacity`, `cmb_released` и историю этих значений для графиков.
- Во время рекомбинации центральное поле теперь плавно переходит от мутной горячей плазмы к прозрачному CMB-отпечатку.
- Checkpoint рекомбинации показывает температуру, масштаб `a(t)`, ионизацию, непрозрачность, `H(a)`, доли radiation/matter и контраст CMB.
- Правая панель UI получила дополнительный график `Рекомбинация` с кривыми ионизации и непрозрачности.
- Timeline и pipeline используют stage id `recombination` вместо технического `recombination_preview`.
- Синхронизирована версия проекта до 0.0.12.

## v0.0.13 -> v0.0.14
